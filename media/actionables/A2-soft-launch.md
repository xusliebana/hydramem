# A2 — Soft launch / semilla (semana 1)

**Objetivo:** llegar al lanzamiento grande con ~50–100 ⭐ base y un pitch ya pulido.
No quieres que el Show HN aterrice en un repo con 3 estrellas.

## Pasos

1. **Pide ⭐ genuinas a 15–30 personas** (amigos, colegas, ex-compañeros).
   - Mensaje honesto: "Acabo de open-sourcear esto, ¿te importa echarle un ojo y, si te
     parece útil, darle star? Feedback honesto bienvenido."
   - ❌ **NUNCA bots ni intercambios de stars** — viola los ToS de GitHub y se detecta.

2. **Post técnico en Dev.to / Hashnode / Medium.**
   - Contenido: [posts/14-devto-technical-blog.md](../posts/14-devto-technical-blog.md)
   - Publícalo en las 3 plataformas (cross-post) con canonical URL a una.
   - Tags: `ai`, `opensource`, `python`, `mcp`.

3. **PRs a 3–4 listas "awesome"** (⭐ pasivas durante meses).
   - Contenido/entrada: [posts/20-awesome-list-pr.md](../posts/20-awesome-list-pr.md)
   - Prioriza `awesome-mcp-servers` (categoría 🧠 Knowledge & Memory).

4. **Publica en directorios MCP** (ver [A8-directories.md](./A8-directories.md)):
   - glama.ai/mcp directory
   - registro oficial MCP

5. **Valida el mensaje.** Mira qué frase engancha en los comentarios de Dev.to / amigos.
   Ajusta el tagline y el primer comentario del Show HN en consecuencia.

## Criterio de "hecho"
- [ ] ~50–100 ⭐ reales
- [ ] 1 post técnico publicado (3 plataformas)
- [ ] 3–4 PRs a listas awesome abiertos
- [ ] Pitch del Show HN ajustado con el feedback recibido
